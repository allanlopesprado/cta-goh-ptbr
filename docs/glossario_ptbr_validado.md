## 12. Notas Técnicas para Tradução

### Tags de Formatação — NUNCA TRADUZIR
- `<f(nome)>` — fonte tipográfica
- `<s(número)>` — tamanho
- `<b>` ou `<b+>` — negrito
- `<c(hex)>` — cor
- `<e(tipo)>` — efeito
- `<m(tipo)>` — modificador
- `%` no início de linha de créditos — separador de segmento

### Estrutura do .pot
```
msgctxt "chave_do_contexto"
msgid "texto original em inglês"
msgstr ""
```
→ Tradução vai em `msgstr "texto traduzido"`

### Cabeçalho do .po
Alterar:
- `Language: en` → `Language: pt_BR`
- `Project-Id-Version:` → manter nome do projeto

### Nomes Próprios de Unidades Históricas — MANTER ORIGINAL
Exemplos: `Leutnant [Heer]`, `Panzergrenadier-Division`, `Waffen-SS`, nomes de tanques, designações de armas, etc.
